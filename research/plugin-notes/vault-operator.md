# Vault Operator

**Pattern:** Sidebar Chat + Event-Driven + Knowledge Graph
**Repo:** pssah4/vault-operator
**Downloads:** ~7,800

## Why It Matters

Most architecturally mature agent plugin. Its three-layer memory, approval workflow, and plugin API discovery are building blocks Pattern 5 needs.

## Key Capabilities

- Agent loop: action → vault execution → result → continue
- Inline chat anchored to text selection (step toward spatial)
- Three-layer memory: session summaries, durable facts, soul profile
- Vault health check: orphans, broken links, weak clusters
- MCP server mode: vault accessible from external agents
- Plugin API discovery: agent invokes other plugins (Excalidraw, Dataview)
- "Every action visible, every write needs approval" — the right UX

## Building Block for Pattern 5

- Three-layer memory → agent persistence model
- Approval workflow → trust model for canvas modifications
- Plugin API discovery → agent could invoke Advanced Canvas events
- Inline chat anchored to text → step toward spatial presence
